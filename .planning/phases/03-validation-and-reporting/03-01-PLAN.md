Implement the following four functions in this order, using the exact patterns from RESEARCH.md:

**1. load_xsd(schema_path: Path) -> 'etree.XMLSchema | None':**
- If schema_path does not exist: print a WARNING to stderr (`f"WARNING: {schema_path} not found — XSD validation will be skipped"`) and return None
- If the file exists but is corrupt/invalid XSD: print `f"ERROR: ALTO XSD is corrupt or invalid: {schema_path}\n  {e}"` to stderr and call sys.exit(1)
- On success: return the compiled etree.XMLSchema object
- Use `etree.parse(str(schema_path))` + `etree.XMLSchema(schema_doc)` (not etree.fromstring)
- Catch `etree.XMLSchemaParseError` for the corrupt case

**2. validate_alto_file(alto_path: Path, xsd: 'etree.XMLSchema | None') -> tuple[bool, 'str | None', list[str]]:**
Returns (schema_valid, schema_error, coord_violations).
- Parse with `etree.parse(str(alto_path))`; catch `etree.XMLSyntaxError` and return `(False, f"XML parse error: {e}", [])`
- XSD validation: only if xsd is not None; use `xsd.validate(tree)` + `xsd.error_log.last_error`; extract `err.message` for schema_error (fallback: "Unknown schema error")
- Call `_check_coordinates(root)` for coord_violations
- If xsd is None: schema_valid = True, schema_error = None (XSD unavailable is treated as skipped, not failed)
- Use `etree.parse()` NOT `etree.fromstring()` — XMLSchema.validate() requires an _ElementTree, not an _Element

**3. _check_coordinates(root: etree._Element) -> list[str]:**
- Namespace constant: `ns = 'http://schema.ccs-gmbh.com/ALTO'`
- Find `Page` element: `root.find(f'.//{{{ns}}}Page')`
- If no Page element: return []
- Extract `page_w = int(page.get('WIDTH', 0))`, `page_h = int(page.get('HEIGHT', 0))`; wrap in try/except (ValueError, TypeError) returning [] on error
- If `page_w == 0 or page_h == 0`: return `['Page WIDTH or HEIGHT is 0 or missing — coordinate check skipped']`
- Iterate `root.iter(f'{{{ns}}}String')`, extract HPOS, VPOS, WIDTH, HEIGHT with int() + try/except (ValueError, TypeError) → continue on error
- Truncate CONTENT to 40 chars for readability: `elem.get('CONTENT', '')[:40]`
- Flag violations:
  - `hpos + width > page_w` → append `f"HPOS+WIDTH={hpos+width} > page_width={page_w} at String '{content}'"`
  - `vpos + height > page_h` → append `f"VPOS+HEIGHT={vpos+height} > page_height={page_h} at String '{content}'"`
- Return violations list (may be empty)

**4. validate_batch(file_records: list[dict], xsd: 'etree.XMLSchema | None') -> tuple[list[dict], int]:**
- For each record in file_records:
  - If `record.get('error_status') != 'ok'`: set `record['schema_valid'] = None`, `record['coord_violations'] = []`, continue
  - If alto_path (`Path(record['output_path'])`) does not exist: set same None/[] defaults, continue
  - Call `validate_alto_file(alto_path, xsd)` → unpack to `schema_valid, schema_error, coord_violations`
  - Set `record['schema_valid'] = schema_valid`, `record['schema_error'] = schema_error`, `record['coord_violations'] = coord_violations`
  - If `not schema_valid or bool(coord_violations)`: set `record['error_status'] = 'ocr_ok, validation_warnings'` and increment warning_count
- Return `(file_records, warning_count)`

Do NOT modify any existing function. Do NOT move any existing code. Insert the new section between `write_error_log()` and `run_batch()` only.
