**JavaScript (inline `<script>` at end of `<body>`):**

Module-level state:
```js
const queue = new Map();   // filename → {filename, stem, state, wordCount}
let runActive = false;
let doneCount = 0;
let totalInRun = 0;
let runStartTime = null;
```

**`stem(filename)`** helper: `filename.replace(/\.[^.]+$/, '').toLowerCase()` — matches `secure_filename` stem normalization in app.py.

**Drop zone setup (`setupDropZone`):**
- `drop-zone` click → `file-input.click()`
- `file-input` change → `handleFiles(e.target.files)`; reset input value after so same file can be re-selected after removal
- `dragenter`/`dragover` on drop-zone → `e.preventDefault(); dropZone.classList.add('dragover')`
- `dragleave`/`drop` → `dropZone.classList.remove('dragover')`
- `drop` → `e.preventDefault(); handleFiles(e.dataTransfer.files)`

**`handleFiles(fileList)`:**
```
for each file:
  if runActive → ignore (button is disabled; user cannot add during run — acceptable UX)
  if not (.tif or .tiff extension, case-insensitive) → addInvalidRow(file.name)
  elif queue.has(file.name) → ignore (silent de-duplication)
  else → queue.set(file.name, {filename: file.name, stem: stem(file.name), state: 'pending', wordCount: null})
       → renderFileRow(file.name)
       → updateStartButton()
```

**`addInvalidRow(filename)`:**
- Append `<li class="file-row state-invalid">` with text `✗ {filename} — Not a TIFF file`
- `setTimeout(() => li.remove(), 3000)`

**`renderFileRow(filename)`:**
- Append `<li id="row-{escapedFilename}" class="file-row state-pending">` with:
  - `<span class="name">{filename}</span>`
  - `<button class="remove" aria-label="Remove" title="Remove">×</button>`
- The × button `onclick` → `removeFile(filename)`

**`removeFile(filename)`:**
- `queue.delete(filename)`
- `document.getElementById('row-' + CSS.escape(filename))?.remove()`
- `updateStartButton()`

**`updateStartButton()`:**
- `btn-start.disabled = runActive || queue.size === 0`
- Label: `runActive ? 'Processing…' : 'Start OCR'`

**`updateRowState(filename, state, {wordCount, errorMsg})`:**
- Find `li#row-{CSS.escape(filename)}`, update class to `state-{state}`, update `.name` content:
  - `state === 'done'`: `<a href="/viewer/${stem(filename)}">${escapeHtml(filename)}</a> (${wordCount ?? '?'} words)` — remove × button
  - `state === 'error'`: `✗ ${escapeHtml(filename)} — ${escapeHtml(errorMsg ?? 'failed')}` — remove × button
  - `state === 'processing'`: leave as-is (CSS prefix handles spinner)
  - `state === 'skipped'`: treat as `done` with wordCount = null, show `(skipped)`

**`startRun()`:**
- POST /upload with all queued files as FormData
- On 200 response: POST /run
- On /run 202 response: open SSE, disable Start button, set `runActive = true`, set `doneCount = 0`, `totalInRun = queue.size`, `runStartTime = Date.now()`
- Mark all rows as `state-processing` (the worker processes sequentially; mark all pending as "queued/processing" at start)
- Open `new EventSource('/stream')` and attach handlers

**SSE handlers:**
- `file_done` event:
  ```js
  const d = JSON.parse(e.data);
  const filename = /* find from queue by matching d.stem === stem(f) */;
  doneCount++;
  // Update queue entry state
  queue.get(filename).state = d.state;
  queue.get(filename).wordCount = d.word_count;
  updateRowState(filename, d.state === 'skipped' ? 'done' : d.state, {wordCount: d.word_count, errorMsg: d.error_msg});
  updateStatusLine();
  ```
  Finding filename from stem: iterate `queue.keys()` and match `stem(filename) === d.stem`.

- `run_complete` event:
  ```js
  sse.close();
  runActive = false;
  statusLine.textContent = '';  // hide status line per user decision
  updateStartButton();
  // Re-enable drop zone for new batch; queue.clear() is NOT called here
  // User must drag new files which clears old results on first new drop
  ```

  On first new file drop after run_complete: `if (!runActive && doneCount > 0)` → `queue.clear(); clearFileList(); doneCount = 0; runStartTime = null;` — then add the new file. This implements "dragging new files clears the old queue and builds a new one."

**`updateStatusLine()`:**
```js
if (!runActive) { statusLine.textContent = ''; return; }
const elapsed = (Date.now() - runStartTime) / 1000;
const eta = doneCount > 0 ? Math.round(elapsed / doneCount * (totalInRun - doneCount)) : '?';
statusLine.textContent = `${doneCount} / ${totalInRun} files done — ETA ${eta}s`;
```

**`clearFileList()`:** `fileList.innerHTML = '';`

**`escapeHtml(str)`:** same helper as viewer.html (replace &, <, >, ").

**Error handling:**
- POST /upload fails (non-200): show a paragraph `Upload failed — ${resp.status}` in status-line, re-enable button
- POST /run fails (409 already running): show `Already running — wait for current batch to finish`
- POST /run fails (400 no pending files): show `No pending files to process`
- SSE connection error: show `Connection lost — reload to retry` in status-line

**Initial page load** (DOMContentLoaded):
- Just set up drop zone; no fetch on load (unlike viewer.html which fetches /files)
- Start button starts disabled (`queue.size === 0`)

CSS: Use flexbox for file rows. No animations beyond CSS transitions. No external libraries. Pure vanilla JS — no bundler. Match the existing viewer.html style (sans-serif, muted palette, minimal).
