- would-process entries go in would_process as filename only (tiff_path.name)
- Print output to stdout in this exact format (CONTEXT.md locked decision):
  ```

    scan_001.tif
    scan_002.tif
  Would skip (N):
    scan_003.tif (output exists)
  Total: N would be processed, M already done
  ```
  Where indentation is 2 spaces. Empty sections must still print their header (e.g. "Would process (0):" with nothing below it).
- Call sys.exit(0) after printing — no report, no error log, no ALTO files written

- --verbose is silently ignored when --dry-run is active (the pre-flight block does not print verbose content; verbose is simply never acted on in this path)
- --force affects the skip-check: with args.force=True, the condition `if not args.force and out_path.exists()` is False for every file, so all TIFFs land in would_process

- Move validate_tesseract() — it must still run before the dry-run gate (dry-run validates the full run would succeed, not just that files exist, per RESEARCH.md open question 3)
