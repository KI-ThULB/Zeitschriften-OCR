Restructure the image panel HTML and implement the zoom/pan engine.

**HTML restructuring** — inside `#image-panel`, wrap `<img id="tiff-img">` and `<svg id="overlay">` together inside a new div `id="image-container"`. The container div receives the CSS transform; the SVG stays inside it so both move together with zero coordinate recalculation.

Current structure:
```html
<div id="image-panel">
  <div id="image-wrapper">
    <img id="tiff-img" ...>
    <svg id="overlay" ...>...</svg>
  </div>
</div>
```

New structure:
```html
<div id="image-panel">
  <div id="image-container">
    <img id="tiff-img" ...>
    <svg id="overlay" ...>...</svg>
  </div>
</div>
```

`#image-panel` CSS: `overflow: hidden; position: relative; cursor: grab; flex: 1; background: #333;`
`#image-container` CSS: `position: absolute; top: 0; left: 0; transform-origin: 0 0;`
Remove the `overflow: auto` that was on `#image-panel` (must be hidden so pan works — scrollbars fight with transform-based pan).

**Zoom/pan state variables** — add at the top of the script block alongside existing state:
```js
let zoomLevel = 1.0;     // current scale factor (1.0 = fit-to-width)
let fitScale = 1.0;      // scale factor that makes image fill panel width
let panX = 0;            // current horizontal pan offset in CSS px
let panY = 0;            // current vertical pan offset in CSS px
const ZOOM_MIN = 0.25;   // 25% of fit-to-width baseline
const ZOOM_MAX = 4.0;    // 400% of fit-to-width baseline
const ZOOM_STEP = 1.1;   // multiply/divide per wheel tick
```

`zoomLevel` is relative to `fitScale` — `zoomLevel=1.0` always means fit-to-width. The CSS scale applied is `fitScale * zoomLevel`.

**`recomputeFitScale()` function** — called whenever the panel size changes or a new file loads. Computes `fitScale = panelWidth / pageWidth` using `pageWidth`/`pageHeight` globals set by loadFile(). Resets `zoomLevel = 1.0`, `panX = 0`, `panY = 0`, then calls `applyTransform()` and `updateResetButton()`. Guard: if `pageWidth === 0` return early.

**`applyTransform()` function**:
```js
function applyTransform() {
  const scale = fitScale * zoomLevel;
  document.getElementById('image-container').style.transform =
    `translate(${panX}px, ${panY}px) scale(${scale})`;
  updateResetButton();
}
```

`translate` before `scale` means panX/panY are in screen pixels (pre-scale) — natural feel for drag panning.

**`clampPan()` function** — enforces soft pan bounds: at least 20% of the image must remain visible. Compute rendered dimensions (`pageWidth * fitScale * zoomLevel`), clamp panX/panY so at most 80% of the image can be dragged off any edge using `Math.min`/`Math.max`.

**`applyZoom(factor, origin)` function** — shared zoom logic for wheel and keyboard:
```js
function applyZoom(factor, origin) {
  if (editingSpan) cancelEdit();
  const newZoom = Math.min(ZOOM_MAX, Math.max(ZOOM_MIN, zoomLevel * factor));
  const panel = document.getElementById('image-panel');
  const cx = origin ? origin.x : panel.clientWidth / 2;
  const cy = origin ? origin.y : panel.clientHeight / 2;
  const scale = fitScale * zoomLevel;
  const imgX = (cx - panX) / scale;
  const imgY = (cy - panY) / scale;
  const newScale = fitScale * newZoom;
  panX = cx - imgX * newScale;
  panY = cy - imgY * newScale;
  zoomLevel = newZoom;
  clampPan();
  applyTransform();
}
```

**Mouse-wheel zoom handler** — attach `wheel` listener to `#image-panel`. Call `e.preventDefault()`. Call `applyZoom(e.deltaY < 0 ? ZOOM_STEP : 1/ZOOM_STEP, {x: e.offsetX, y: e.offsetY})`.

**Click-drag pan handler** — mousedown/mousemove/mouseup on `#image-panel`. State vars: `let isDragging = false; let dragStartX = 0; let dragStartY = 0; let dragStartPanX = 0; let dragStartPanY = 0; let didPan = false;`

- `mousedown`: record drag start positions, set `isDragging = true`, `didPan = false`, add class `dragging` to `#image-panel` (sets grabbing cursor via CSS).
- `mousemove`: if `isDragging`, compute `dx = e.clientX - dragStartX`, `dy = e.clientY - dragStartY`. If `!didPan && Math.hypot(dx,dy) > 5`: set `didPan = true`, call `cancelEdit()` if `editingSpan` is set. If `didPan`: `panX = dragStartPanX + dx`, `panY = dragStartPanY + dy`, call `clampPan()`, call `applyTransform()`.
- `mouseup` / `mouseleave`: set `isDragging = false`, remove class `dragging` from `#image-panel`.

**Suppress post-pan image click** — wrap the `highlight-rect` click listener body: `if (didPan) { didPan = false; return; }` at the top.

**Wire into `loadFile()`** — call `recomputeFitScale()` after `pageWidth = data.page_width` and `pageHeight = data.page_height` are assigned inside the ALTO fetch callback.

**Wire ResizeObserver** — in the existing `setupResizeObserver()`, add `recomputeFitScale()` before `showHighlight(activeWord)` so fit-to-width recalculates on window resize.

**`updateResetButton()` function** — creates or updates an overlay button inside `#image-panel`:
```js
function updateResetButton() {
  let btn = document.getElementById('zoom-reset-btn');
  if (!btn) {
    btn = document.createElement('button');
    btn.id = 'zoom-reset-btn';
    document.getElementById('image-panel').appendChild(btn);
    btn.addEventListener('click', (e) => {
      e.stopPropagation();
      recomputeFitScale();
    });
  }
  const isDefault = (Math.abs(zoomLevel - 1.0) < 0.01);
  btn.style.display = isDefault ? 'none' : '';
  if (!isDefault) {
    const pct = Math.round(fitScale * zoomLevel * 100);
    btn.textContent = `${pct}% \u22A1`;
  }
}
```

`\u22A1` is the squared-dot character used as a fit/reset icon. `applyTransform()` already calls `updateResetButton()` (added in Task 1). Also call `updateResetButton()` at the end of `recomputeFitScale()`.
