**pipeline.py — new function `deskew_image()` (insert between load_tiff() and detect_crop_box())**:

Implement exactly as specified in the research doc (04-RESEARCH.md, Pattern 1). The function signature and behavior:

```python
def deskew_image(
    image: Image.Image,
    max_angle: float = DESKEW_MAX_ANGLE,
) -> tuple[Image.Image, 'float | None', bool]:
    """Detect and correct scan skew using Hough line transform.

    Args:
        image: PIL Image of the scan (any mode; converted to grayscale internally)
        max_angle: Maximum plausible correction angle in degrees (default: DESKEW_MAX_ANGLE).
                   Angles above this are treated as detection failures.

    Returns:
        corrected: PIL Image — rotated if angle is plausible, original otherwise
        angle: Detected angle in degrees (float), or None if detection failed
        fallback_used: True if original image is returned (detection failed OR
                       angle exceeded max_angle)
    """
```

Implementation rules (all from 04-RESEARCH.md):
1. Convert to grayscale numpy: `gray = np.array(image.convert('L'))`
2. Wrap `determine_skew(gray)` in a bare `except Exception` — return `(image, None, True)` if it raises
3. If `angle is None` → return `(image, None, True)`
4. If `abs(angle) > max_angle` → return `(image, angle, True)` (detected but implausible)
5. If `abs(angle) < 0.05` → return `(image, angle, False)` (near-zero: skip rotation to avoid resampling artifact — Success Criterion 4)
6. Otherwise rotate: detect `image.mode` for fill color: `fill = 255 if image.mode == 'L' else (255, 255, 255) if image.mode == 'RGB' else 255`
7. Rotate with `image.rotate(angle, expand=True, fillcolor=fill, resample=Image.Resampling.BICUBIC)` — NOT `Image.BICUBIC` (deprecated)
8. Return `(corrected, angle, False)`
