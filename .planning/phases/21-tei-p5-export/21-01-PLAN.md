<feature>
  <name>TEI P5 XML builder — tei.py</name>
  <files>tei.py, tests/test_tei.py</files>
  <behavior>
    build_tei(output_dir: Path, stem: str) -> bytes

    Reads:
      output_dir/alto/<stem>.xml    — ALTO 2.1 (CCS-GmbH namespace, same as mets.py)
      output_dir/segments/<stem>.json  — VLM article regions (optional; fallback if absent)

    Returns UTF-8 TEI P5 XML bytes with:
      - <teiHeader> with fileDesc (titleStmt/title = stem, publicationStmt, sourceDesc) and
        encodingDesc with projectDesc mentioning "Zeitschriften-OCR" and today's date (ISO)
      - <facsimile> with one <surface xml:id="page-{stem}" ulx="0" uly="0" lrx="{pw}" lry="{ph}"
        facs="../uploads/{stem}.tif"> and one <zone xml:id="zone-{stem}-{region.id}"
        ulx/uly/lrx/lry in ALTO pixel coords, type={region.type}> per VLM region
      - <text><body> with <pb n="1" facs="#page-{stem}"/> then one <div type="article" n="{i+1}">
        per VLM region; inside each div, a <head> if region has title, then <p> elements per
        detected paragraph; words column-sorted then hyphens rejoined; <lb/> between lines
        (not after last line of each <p>)
      - No VLM case: one <div type="article"> wrapping all paragraphs; XML comment at top of
        <body>: <!-- VLM segmentation absent: all text rendered as single article -->

    Cases to cover in tests:
      - Single TextBlock, two TextLines, two words per line → two <lb/> boundaries visible, last line no lb
      - End-of-line hyphen "Ver-" + "bindung" → "Verbindung" in output, intermediate lb suppressed
      - Two-column layout (blocks at HPOS 0 and HPOS 2100 on 4000px page) → left column words before right column words
      - VLM region present → <div type="article" n="1"> + <zone> with correct pixel coords
      - No segments file → one <div type="article"> + XML comment in body + no <zone> elements
      - Repeated calls return identical bytes (idempotent)
      - XML namespace is http://www.tei-c.org/ns/1.0 on root element
      - xml:id on <surface> is "page-{stem}" (no hash); facs on <pb> is "#page-{stem}" (with hash)
      - Zone coordinates use ALTO page_width/page_height from <Page WIDTH= HEIGHT=>; NOT jpeg dimensions
  </behavior>
  <implementation>
    File: tei.py — structure mirrors mets.py exactly:
      1. Module docstring + imports (lxml.etree, json, datetime, pathlib.Path, pipeline for ALTO21_NS)
      2. Namespace constants: TEI_NS, XML_NS, ALTO21_NS (imported from pipeline), MODS_NS
      3. Helper _t(tag) → Clark notation in TEI_NS; _xml_id(val) → Clark notation for xml:id
      4. _read_mods_metadata(output_dir, stem) → dict with title/date/publisher (optional METS read, always falls back)
      5. _column_sort(all_strings, page_width) → list sorted by column cluster then VPOS then HPOS
         - Group TextBlock elements by HPOS; gap > 0.2 * page_width separates columns
         - Full-width blocks (WIDTH > 0.6 * page_width) sort first
         - Within each column: sort by block VPOS ascending, then word VPOS ascending
         - Input: list of dicts {content, hpos, vpos, width, height, block_hpos, block_width, block_vpos, line_end}
      6. _rejoin_hyphens(words) → list with hyphen-split pairs merged
         - If word content ends with '-' and line_end=True: concatenate content with next word content (strip trailing '-')
         - Merged word inherits line_end=False (the intermediate lb is suppressed)
         - If the pair was at the actual end of its paragraph, the merged word retains the final state
      7. _assign_word_to_regions(word_hpos, word_vpos, word_w, word_h, regions, page_width, page_height) → region id or None
         - Same overlap test as mets.py _find_word_ids_in_region(): hpos < hpos_max and (hpos+w) > hpos_min and vpos < vpos_max and (vpos+h) > vpos_min
      8. build_tei(output_dir, stem) → bytes
         - Read ALTO: etree.parse, materialise all_strings = list(root.iter(...String...)) (proxy recycling fix)
         - Read segments: json.loads (or empty regions=[])
         - Build column-sorted, hyphen-rejoined word list with line_end flags
         - Assign words to regions; words with no region go to fallback bucket
         - Construct lxml tree: TEI > teiHeader > facsimile > text
         - Return etree.tostring(root, xml_declaration=True, encoding='UTF-8', pretty_print=True)

    LINE_END FLAG — detect last String in each TextLine:
    Materialise all TextLines: `all_lines = list(root.iter(f'{{{ALTO21_NS}}}TextLine'))`.
    For each TextLine, the last String child is line-terminal.

    When iterating all_strings, check if (s.get('CONTENT'), s.get('HPOS'), s.get('VPOS')) is in line_end_set.

    COLUMN SORT (_column_sort):
    - Extract block for each string: walk TextBlock elements, build string→block mapping
    - Group blocks by HPOS cluster: sort blocks by HPOS; a gap > 0.2*page_width starts a new column group
    - Full-width blocks (block WIDTH > 0.6*page_width): column index = -1 (sorts first)
    - Within each column: sort by block VPOS ascending, then string VPOS ascending, then HPOS ascending
    - Return the flat sorted word list

    HYPHEN REJOIN (_rejoin_hyphens):
    - Walk words in order; if word['content'].rstrip().endswith('-') and word['line_end']:
      - Pop next word, concatenate: merged_content = word['content'].rstrip()[:-1] + next_word['content']
      - Merged word: line_end=False (intermediate lb suppressed), content=merged_content
      - Do NOT emit lb for this boundary

    MIXED CONTENT (lb elements) — follow Research Pattern 3 exactly:
    - For each <p>, iterate words. Accumulate text. When line_end=True and NOT the last word in the group:
      - Flush accumulated text to p_elem.text or last child's .tail
      - Insert <lb/> element with lb.tail = ''
      - Reset accumulator
    - After loop: flush remaining text (no lb after final word)

    VLM REGION MAPPING:
    - Load output_dir/segments/<stem>.json; if file absent or empty regions, use regions=[]
    - For each region, compute pixel bbox: ulx=int(bb['x']*pw), uly=int(bb['y']*ph), lrx=int((bb['x']+bb['width'])*pw), lry=int((bb['y']+bb['height'])*ph)
    - Add <zone xml:id="zone-{stem}-{region['id']}" ...> to <surface>
    - Assign each word to the region whose pixel bbox it overlaps (same test as mets.py); unassigned words go to fallback
    - No VLM data: body comment "VLM segmentation absent: all text rendered as single article", one div, no zones

    SURFACE facs attribute:
    - Use `facs=f'../uploads/{stem}.tif'` (files are in output/uploads/ per app.py UPLOAD_SUBDIR pattern)

    TEIHEADER title:
    - Try _read_mods_metadata(); if title is None, use stem (filename-derived fallback, always triggered since no per-stem METS files on disk)
