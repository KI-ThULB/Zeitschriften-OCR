  <action>
Add the following pure functions to viewer.html inside the `<script>` block, after the existing `normalizeWords`/`rejoinHyphens`/`columnSort` functions and before `renderWords()`. Do NOT modify `renderWords()` in this task — that is Plan 02.

1. Add the `VLM_TYPE_TO_ROLE` constant:
   ```javascript
   const VLM_TYPE_TO_ROLE = {
     headline:      'heading',
     article:       'paragraph',
     advertisement: 'advertisement',
     illustration:  'paragraph',
     caption:       'caption',
   };
   ```

2. Add `detectParagraphs(block, wordById)`:
   - Gathers all words for the block using `(block.word_ids || []).map(id => wordById[id]).filter(Boolean)`
   - Sorts by `vpos` ascending
   - Guard: if `blockWords.length === 0`, return `[{ block, words: [] }]`
   - Compute unique VPOS values: `[...new Set(blockWords.map(w => w.vpos))].sort((a, b) => a - b)`
   - Guard: if `vposValues.length <= 1`, return `[{ block, words: blockWords }]` (single paragraph, no split possible)
   - Compute gaps between successive unique VPOS values; sort gaps; take median (`sorted[Math.floor(sorted.length / 2)]`)
   - Threshold = `median * 1.5` (per CONTEXT.md locked decision)
   - Walk `blockWords` in VPOS order; start new paragraph group when gap from last word's VPOS to current word's VPOS exceeds threshold
   - Return array of `{ block, words: [...] }` objects — one per detected paragraph within the TextBlock
   - Note: `vpos` comes from the word object's `vpos` field already present in `data.words` response

3. Add `intersectionArea(ax1, ay1, ax2, ay2, bx1, by1, bx2, by2)`:
   - Standard axis-aligned rectangle intersection: `ix1=max(ax1,bx1)`, `iy1=max(ay1,by1)`, `ix2=min(ax2,bx2)`, `iy2=min(ay2,by2)`
   - Return `(ix2-ix1)*(iy2-iy1)` when positive, else `0`

4. Add `assignRoles(blocks, articles, pageWidth, pageHeight)`:
   - `articles` is the `currentArticles` array (regions from VLM, each with `{id, type, bounding_box:{x,y,width,height}}`)
   - For each block in `blocks`, find the VLM region with maximum intersection area
   - Convert VLM bounding box to ALTO pixel space: `rx1 = bb.x * pageWidth`, `ry1 = bb.y * pageHeight`, `rx2 = rx1 + bb.width * pageWidth`, `ry2 = ry1 + bb.height * pageHeight`
   - CRITICAL: use `pageWidth`/`pageHeight` (ALTO page units), NOT jpeg dimensions — see Research pitfall 1
   - TextBlock rect: `block.hpos, block.vpos, block.hpos + block.width, block.vpos + block.height`
   - Best role = `VLM_TYPE_TO_ROLE[region.type] || 'paragraph'` for max-overlap region
   - Default role = `'paragraph'` when no overlap or `articles` is empty
   - Returns `Map<block.id, role>`

5. Add `buildParaBlocks(blocks, wordById, articles, pageWidth, pageHeight)`:
   - Call `assignRoles(blocks, articles, pageWidth, pageHeight)` → `roleMap`
   - For each block in `blocks`: get role from `roleMap.get(block.id) || 'paragraph'`; call `detectParagraphs(block, wordById)` → `subParas`; for each `{ words }` in subParas, push `{ block, role, words }` to result array
   - After processing all blocks, find orphan words: words in `wordById` whose `id` is not in any block's `word_ids`. Collect as `orphans`; if `orphans.length > 0`, push `{ block: null, role: 'paragraph', words: orphans }` as the final entry
   - Return the result array

Exact code patterns from RESEARCH.md are authoritative — follow them. Apply the guards for single-word blocks and empty blocks as specified.
