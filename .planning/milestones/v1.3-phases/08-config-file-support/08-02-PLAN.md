**Step A: Add pre-parser block at the very top of main(), before the existing `parser = argparse.ArgumentParser(...)` line.**

Insert this block:

```python
# Step 0: Pre-parse to extract --config before the full parse.
# parse_known_args() silently ignores all other flags — this only captures --config.
pre = argparse.ArgumentParser(add_help=False)
pre.add_argument('--config', type=Path, default=None)
pre_args, _ = pre.parse_known_args()

# Load config and build the validated dict BEFORE constructing the full parser,
# so set_defaults() can inject values before parse_args() runs.
config_values: dict = {}
if pre_args.config is not None:
    config_values = load_config(pre_args.config)  # exits on error (OPER-05)
```

**Step B: Add --config to the main parser's add_argument() block.**

After the existing `parser.add_argument('--verbose', ...)` call (the last add_argument currently in main()), add:

```python
parser.add_argument(
    '--config',
    type=Path,
    default=None,
    metavar='PATH',
    help='JSON file of flag defaults. CLI flags always override config values. '
         'Keys must match argparse dest names (e.g. "lang", "dry_run").',
)
```

**Step C: Add set_defaults() injection immediately after the `--config` add_argument call, before the existing `args = parser.parse_args()` line.**

```python
# Inject config values as new defaults — CLI flags (parse_args) take precedence.
# Precedence: explicit CLI flag > set_defaults() value > add_argument() default
if config_values:
    parser.set_defaults(**config_values)
```

**Step D: Leave `args = parser.parse_args()` unchanged** — it now runs with config values already injected as defaults, so CLI flags automatically win without any additional logic.

**Step E: Add verbose config summary block after the existing `validate_tesseract(args.lang)` call and before the `if not args.input.is_dir():` check.**

The exact insertion point in main() (from pipeline.py line ~944):

```python
    validate_tesseract(args.lang)

    # NEW — Phase 8: verbose config summary (OPER-04)
    if args.verbose and args.config is not None and config_values:
        parts = []
        for key, file_val in config_values.items():
            cli_val = getattr(args, key)
            if cli_val != file_val:
                parts.append(f"{key}={file_val} \u2192 {cli_val} (CLI override)")
            else:
                parts.append(f"{key}={cli_val}")
        print(f"Config: {', '.join(parts)} (from {args.config.name})")

    # CLI-01: Validate --input is a directory
    if not args.input.is_dir():
```

Use the Unicode arrow `→` (U+2192) in the override format, matching the CONTEXT.md decision verbatim: `lang=deu → eng (CLI override)`.

**Why after validate_tesseract():** If Tesseract is missing, the run aborts immediately and the config summary is noise. Printing after validate_tesseract() ensures it only appears when the run will actually proceed. This matches the locked decision "before any file processing" while keeping output sensible.

**Nothing else in main() changes.** The existing `n_workers`, `load_xsd()`, `args.input.is_dir()` check, `args.output.mkdir()`, `validate_only` path, `dry_run` path, and `run_batch()` call are all untouched. The `config_values` dict and `args.config` Path are both available in main() scope for the summary block.
