**Step A — Modify run_ocr() to accept capture_output parameter**

Change the signature from:
```python
def run_ocr(image: Image.Image, lang: str = 'deu', psm: int = 1, dpi: int = 300) -> bytes:
```
To:
```python
def run_ocr(image: Image.Image, lang: str = 'deu', psm: int = 1, dpi: int = 300, capture_output: bool = False) -> tuple[bytes, str]:
```

Return type changes from `bytes` to `tuple[bytes, str]` — the second element is the captured Tesseract stdout+stderr (empty string when capture_output=False).

Implementation:
- When capture_output=False: call pytesseract.image_to_alto_xml() as today. Return `(alto_bytes, '')`.
- When capture_output=True: write the PIL Image to a NamedTemporaryFile as PNG, then run Tesseract directly via subprocess.run() to capture its output AND also get the ALTO XML:

```python
import subprocess
import tempfile

if capture_output:
    with tempfile.NamedTemporaryFile(suffix='.png', delete=False) as tmp:
        tmp_path = tmp.name
    try:
        image.save(tmp_path)
        # Run tesseract to get ALTO XML (stdout output)
        proc = subprocess.run(
            ['tesseract', tmp_path, 'stdout', '--psm', str(psm), '--dpi', str(dpi),
             '-l', lang, 'alto'],
            capture_output=True,
            text=False,
        )
        alto_bytes = proc.stdout if proc.stdout else b''
        tess_text = (proc.stderr.decode('utf-8', errors='replace')).strip()
        if not alto_bytes:
            # fallback: Tesseract wrote to stderr with error; re-raise by using normal path
            alto_bytes = pytesseract.image_to_alto_xml(image, lang=lang, config=config)
            if not isinstance(alto_bytes, bytes):
                alto_bytes = alto_bytes.encode('utf-8')
    finally:
        import os as _os
        _os.unlink(tmp_path)
    return alto_bytes, tess_text
```

Wait — Tesseract's `stdout` output target writes ALTO XML to stdout when `alto` extension is requested, but only with `tesseract FILE stdout -l LANG --psm N --dpi D alto` syntax. The `config` string used by pytesseract contains `--psm N --dpi N` as a single string; when calling subprocess directly, these must be passed as separate arguments. Reconstruct them:

```python
proc = subprocess.run(
    ['tesseract', tmp_path, 'stdout',
     '-l', lang,
     '--psm', str(psm),
     '--dpi', str(dpi),
     'alto'],
    capture_output=True,
)
alto_bytes = proc.stdout if proc.stdout else b''
tess_text = proc.stderr.decode('utf-8', errors='replace').strip()
```

Return `(alto_bytes, tess_text)`.

When capture_output=False, return `(pytesseract.image_to_alto_xml(...) normalised to bytes, '')`.

Add `import subprocess` to the top-level imports if not already present (check: subprocess is already in stdlib but may not be explicitly imported in pipeline.py — add it to the import block if missing).
